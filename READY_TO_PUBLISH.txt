Container:     Dockerfile
Documentation: ARCHITECTURE.md, DISTRIBUTION.md

What it does:
  - Cryptographic proof service
  - Ed25519 signing with private keys
  - Merkle tree append-only audit log
  - Flask REST API on port 5000

To publish:
  docker login                                   # Requires Docker Hub account
  docker push vigil/proof-service:0.1.0
  docker push vigil/proof-service:latest

Deployment (after publishing):
  docker run -d -p 5000:5000 vigil/proof-service:0.1.0
  curl http://localhost:5000/health

NOTES

1. Both packages can be published independently:
   - CLI can be used without Docker (points to remote service)
   - Service can run standalone (Docker only)

2. Version numbers should stay in sync:
   - vigil-cli v0.1.0
   - vigil/proof-service:0.1.0

3. For future updates:
   - Update version in setup.py and pyproject.toml
   - Rebuild package: python -m build
   - Rebuild image: docker build -t vigil/proof-service:X.Y.Z .
   - Publish both together for compatibility

4. Security:
   - CLI is open source (users should review)
   - Service holds private keys (deploy in secure location)
   - Clear trust boundary between them
